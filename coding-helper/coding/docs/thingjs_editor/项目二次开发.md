# **项目结合方式**

基于编辑器创建的项目与 ThingJS 项目本质上是相同的，输出的资源、功能与一般的 ThingJS 项目也基本一致。在使用编辑器开发项目时，您可以根据具体需求选择不同程度的结合方式：

* **资源工具**：编辑器仅用作资源的生产、整合和测试工具，例如组合场景、单独开发和测试预制件及插件。资源输出后，其余开发工作由您自行完成。
* **结合使用**：除了进行资源组织外，部分功能可以由编辑器开发，资源将在编辑器内部管理。其他功能则在编辑器外部自行开发，并与编辑器的项目结合使用。
* **完全使用**：完全依赖编辑器进行开发，项目输出后直接进行部署。

![](https://pan.uino.cn/files/image/2024/9/9ccea214bd078bf2b403dd03317a6d80.png)

关于 “结合使用” 的一些典型的使用场景：

1. 3D 部分的开发可以放在编辑器内，可以通过编写插件或绘制蓝图来实现该部分的业务逻辑，而与界面相关的内容您可以放到编辑器之外的独立开发工程中进行开发和测试，最后进行联调。
2. 或者 3D 部分的预制件、插件等相对独立的部分，都可以在编辑器内进行开发、测试，而项目定制化比较强的逻辑部分，以及和界面结合的部分，可以选择在外面独立工程中进行。
3. 在创建项目时，您可以基于一个项目模板构建项目，一些常用的功能在项目模板中已经实现了，可以通过修改配置、修改资源进行使用，但有项目定制的功能还需要自行扩展，这时可以在外面独立工程中进行开发，并结合这个项目模板进行使用。

您可以灵活运用编辑器和独立开发工程的组合方式，具体实现不局限于上述方式。您可以根据项目需求和团队协作的实际情况，选择更适合的开发流程。

接下来主要讨论如何在编辑器之外的独立工程中加载编辑器项目，并进行二次开发。

> 说明：关于插件开发的详细信息，请参考“插件”章节及“插件开发”文档。

# **加载项目**

在您完成编辑器中的项目后，可以通过 ThingJS 2.0 代码开发来加载该项目。

假设您的独立工程位于`your-project`文件夹中，而编辑器项目存放在`sample01`文件夹内，您可以将编辑器项目放置于独立工程文件夹下，并与测试页面（如`test.html`）并列放置，结构示例如下：

    ```bash
    ├─ your-project     # 您的独立工程文件夹
    │   ├─ sample01     # 编辑器的项目文件夹
    │   │   ├─ assets
    │   │   ├─ libs
    │   │   │   └─ thing.utils.min.js   # ThingJS 引擎工具脚本
    │   │   └─ ......
    └─  └─ test.html    # 测试页面
    ```

在 `test.html` 页面中，您只需引入项目文件夹内的 `thing.utils.min.js` 文件，并通过 `THING.Utils.init` 接口初始化项目路径。下面的 HTML 页面的示例可供参考：

```html
<!DOCTYPE html>
<html lang="">
  <head>
    <meta charset="utf-8" />
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />
    <meta name="viewport" content="width=device-width,initial-scale=1.0" />
    <title>dev-template</title>
    <script src="./samples01/libs/thing.utils.min.js"></script>
    <style>
      * {
        margin: 0;
        padding: 0;
        outline: none;
      }
      html,
      body {
        width: 100%;
        height: 100%;
        overflow: hidden;
      }
      #rootDiv {
        height: 100%;
        position: relative;
      }
    </style>
  </head>
  <body>
    <div id="rootDiv"></div>
  </body>
  <script type="module">
    THING.Utils.init({
      url: './samples01',
      container: document.getElementById('rootDiv'),
      authUrl: 'https://www.thingjs.org.cn/auth/login'
    })
  </script>
</html>
```

启动 WEB 服务并访问该页面，即可查看结果。

# **同步编辑项目**

此时，您可以使用编辑器打开 `sample01` 项目继续进行编辑。在编辑器中保存更改后，请清除浏览器缓存并刷新您的页面（`test.html`），即可立即查看所做的修改。这种方式允许用户一边在编辑器中进行调整，一边在独立工程的页面中查看最新的成果：

![](https://pan.uino.cn/files/image/2024/9/32d7db506f9dab6ef9769dcc142ebd76.png)

> ***注意**：如果加载编辑器输出后的文件夹，也可以看到运行结果。但区别在于，导出后的项目仅包含已经打包和编译的资源（如插件），无法在编辑器中进行进一步编辑。*

# **交互调用**

在您的独立工程中，您可以自由引用或开发其他功能。例如，您可以根据项目需求自行编写界面部分的代码，并通过标准的 ThingJS 事件与编辑器项目中的蓝图进行通信。当用户点击界面时，可以触发事件，蓝图将接收到该事件并执行后续逻辑；反之，蓝图中也可以触发事件，界面则可以接收并展示相应的数据。具体实现和细节可参考 ThingJS 文档的 [蓝图章节](https://wiki.uino.com/book/thingjs-api20/thingjs-documentation-blueprint.html)中关于 “蓝图通信” 的部分。

您同样可以正常调用 ThingJS 提供的接口进行各种操作，比如查询对象、调用对象接口等，无论这些对象是由编辑器创建的，还是在您的独立工程中创建，操作方式都是一样的。如果编辑器项目中使用了某些 “运行时” 插件，也可以在您的独立工程中直接调用这些插件的功能接口。

总之，在您的独立工程中加载 “编辑器项目” 后进行的 ThingJS 开发，与直接引用 ThingJS 库文件进行开发是完全相同的，确保您可以无缝地使用所有功能和接口。
