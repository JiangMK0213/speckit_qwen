## 概述

**ThingJS Editor** 是一款高度可扩展的编辑器框架，其核心功能依托插件实现。通过开发插件，您可以在**编辑时**与**运行时**扩展编辑器功能，以满足各类业务场景的需求。

## 插件创建

### 创建流程

在编辑器中打开「开发 - 插件管理」菜单，选择「创建插件」，填写相关信息后，系统会在项目的插件目录下自动生成基础代码结构，包括入口文件和配置文件。

### 开发语言

插件支持多种开发语言与框架，以满足不同团队的技术栈与开发习惯：

* **JavaScript**：便于快速开发与验证原型。
* **TypeScript**：提供类型检查和更高的代码可靠性。
* **Vue**：适合构建复杂的交互界面。

### 存储位置

插件统一存放在项目根目录下的 `plugins` 文件夹中。示例结构如下：

```bash
[项目路径]/plugins
├── my-plugin
│   ├── index.js     // 编辑时入口文件
│   ├── main.js      // 运行时入口文件
│   ├── config.json  // 插件配置文件
```

### 配置文件

插件信息在`config.json`中初始化：

```bash
{
  "name": "my-plugin",
  "title": "我的插件",
  "version": "1.0.0",
  "description": "插件描述",
  "editor": {
    "entrance": "index.js"
  },
  "runtime": {
    "entrance": "main.js",
    "includes": ["libs/"]
  },
  "params": {}
}
```

> 命名规范：建议遵循 npm 包的命名格式（如 `@team-name/plugin-name`），以便统一管理与识别。

## 生命周期

插件开发中有三个关键的生命周期钩子，方便在合适的时机执行初始化和清理操作：

* **onBeforeInstall**：插件安装前触发，适合执行资源注册或预加载操作。
* **onInstall**：插件安装完成后触发，用于初始化插件的核心逻辑与界面。
* **onUninstall**：插件卸载完成时触发，用于清理资源和事件监听器等。

> **注意**：所有插件的 `onInstall` 都会在所有插件的 `onBeforeInstall` 完成后执行。

## 编辑时插件

编辑时插件用来扩展编辑器本身的功能，例如处理特定资源格式、增加定制菜单、工具栏或面板界面。下面是一个典型的编辑时插件入口文件示例：

```javascript
// index.js
import App from './App.vue'

// 插件安装前
const onBeforeInstall = (params, { app, vm }) => { 
  console.log('onBeforeInstall', app, vm)
}

// 插件安装后
const onInstall = (params, { app, vm }) => {
  console.log('onInstall')
}

// 插件卸载
const onUninstall = ({ app, vm }) => {
  console.log('onUninstall')
}

// 导出组件与生命周期钩子
export default {
  App,
  onBeforeInstall,
  onInstall,
  onUninstall
}
```

### 全局 editor 对象

在编辑时，ThingJS Editor 提供一个全局 `editor` 对象，开发者可以通过该对象访问和扩展编辑器的核心功能，包括但不限于：

* **菜单管理**：添加或修改自定义菜单项
* **工具栏扩展**：增加交互工具
* **布局管理**：创建、移除或调整面板布局
* **文件操作**：读写文件、监听目录变化
* **历史记录**：实现撤销、重做功能

以下是一个在 Vue 组件中使用 `editor` 对象的示例：

```vue
<!-- App.vue -->
<template>
  <div ref="myTemplate">我的面板内容</div>
</template>

<script setup>
import { onMounted, onUnmounted, ref } from 'vue'

const myTemplate = ref()

onMounted(() => {
  // 添加自定义面板
  editor.addPanel({
    id: 'MyPanel',
    name: '我的面板',
    template: myTemplate
  })
})

onUnmounted(() => {
  // 移除自定义面板
  editor.removePanel('MyPanel')
})
</script>
```

## 运行时插件

运行时插件用于为最终应用提供功能扩展。例如，您可以在运行时创建自定义对象、实现高级逻辑或提供可在蓝图中使用的节点。运行时插件与 ThingJS 2.0 中的插件机制一致。以下是一个运行时插件的示例：

```javascript
// main.js
class MyPlugin extends THING.BasePlugin {
  constructor(params) {
    super()
    this.box = null
  }
  // 安装之前
  onBeforeInstall() {
  	console.log('onBeforeInstall')
  }
  // 插件安装完成
  onInstall(options) {
    // 创建一个Box
    this.box = new THING.Box()
  }
  // 插件卸载完成
  onUninstall() {
    this.box.destroy()
    this.box = null
  }
  // 插件方法
  sayHello() {
    console.log('hello MyPlugin!')
  }
}

// 导出插件类
export { MyPlugin }
```

## 依赖管理

### 第三方依赖

插件的第三方依赖可安装在插件目录下或项目根目录下的 `node_modules` 中：

```bash
├── plugins              
│   ├── my-plugin-01      
│   │   ├── node_modules  // 插件独立依赖
├── node_modules          // 多插件共享依赖
```

> **建议**：插件独有的依赖放在插件目录下，共享依赖安装在项目根目录，利于依赖版本的统一管理。

### 内置依赖

无需安装即可直接使用的内置库包括：

* **Vue3**
* **Element-Plus**（包含 icons）
* **Echarts**

```javascript
import { ref } from 'vue'
import { ElButton } from 'element-plus'
import { Close } from '@element-plus/icons-vue
```

### 全局依赖

在项目 `libs/custom` 中放置标准 UMD 格式的 JS 库文件，编辑器会自动加载。这样开发者可在编辑和运行时通过全局变量直接使用这些库。

## 资源引用

在加载图片、JSON 或其他资源文件时，可利用 `import.meta.url` 和 `URL` 对象构造资源的绝对路径，从而避免硬编码路径带来的环境适配问题：

```javascript
// 使用相对于当前模块文件的相对路径
const resourceUrl = new URL('./assets/example.png', import.meta.url).href
```

## 编辑器框架基础

ThingJS Editor 提供了丰富的基础功能与 API，支持开发者灵活地创建和管理插件，扩展编辑器的行为与界面。这些功能涵盖了面板管理、文件操作、历史记录、事件系统等，为插件开发提供了强大的支撑，使插件能够与编辑器无缝集成，并协同工作。完整 API 可查看 [Editor API](https://wiki.uino.com/book/thingjs-editor/66f117533b407c4754cd53cf.html)。

### 1\. 面板操作

支持自定义面板的创建、管理和删除。开发者可以根据需求定制界面布局，为插件提供功能入口和操作界面。

详细文档：[面板操作](https://wiki.uino.com/book/thingjs-editor/6710bf6008b62209a0b23ea0.html)

### 2\. 内置面板

编辑器预置了多种内置面板，开发者可以利用这些面板来管理资源、编辑属性、构建层级结构、编写代码及实现蓝图逻辑。

* **资源面板 (Resource)**：[详细文档](https://wiki.uino.com/book/thingjs-editor/66f11caa3b407c4754cd54a5.html)
* **属性面板 (Inspector)**：[详细文档](https://wiki.uino.com/book/thingjs-editor/66f11c493b407c4754cd546b.html)
* **层级面板 (Hierarchy)**：[详细文档](https://wiki.uino.com/book/thingjs-editor/66f11c123b407c4754cd544c.html)
* **代码面板 (Code)**：[详细文档](https://wiki.uino.com/book/thingjs-editor/66f11b9f3b407c4754cd5413.html)
* **蓝图面板 (Blueprint)**：[详细文档](https://wiki.uino.com/book/thingjs-editor/66f11b8a3b407c4754cd5403.html)
* **场景面板 (Scene)**：[详细文档](https://wiki.uino.com/book/thingjs-editor/66f11cd33b407c4754cd54b5.html)

### 3\. 文件操作与监听

提供文件的读写功能，并支持实时监听目录变化。

详细文档：[文件操作](https://wiki.uino.com/book/thingjs-editor/66f11cf93b407c4754cd54c2.html) | [文件监听](https://wiki.uino.com/book/thingjs-editor/6711d31d08b62209a0b2414b.html)

### 4\. 历史记录

支持操作的撤销与重做。通过历史记录功能，开发者可以将复杂的操作封装为命令，确保操作的可控性与灵活性。

详细文档：[历史记录](https://wiki.uino.com/book/thingjs-editor/66f11bb33b407c4754cd541f.html)

### 5\. 系统菜单

支持自定义系统菜单项，并与编辑器内置菜单（如文件、编辑、开发、面板和帮助）集成，满足个性化功能需求。

详细文档：[系统菜单](https://wiki.uino.com/book/thingjs-editor/66f11c643b407c4754cd5478.html)

### 6\. 工具栏与快速访问栏

* **工具栏**：为场景面板添加按钮和交互控件，方便快捷地触发操作。
* **快速访问栏**：提供常用功能的快速入口，提高场景编辑效率。

详细文档：[工具栏](https://wiki.uino.com/book/thingjs-editor/66f11d383b407c4754cd54de.html) | [快速访问栏](https://wiki.uino.com/book/thingjs-editor/66f11c8e3b407c4754cd5492.html)

### 7\. 面板布局

支持灵活配置和自定义面板布局，开发者可通过调整布局模式优化用户体验，实现多场景适配。

详细文档：[面板布局](https://wiki.uino.com/book/thingjs-editor/66f11b733b407c4754cd53f6.html)

### 8\. 插件事件

提供事件系统支持插件之间的通信，开发者可以灵活地发送、监听和处理事件，实现插件间的高效协作。

详细文档：[事件系统](https://wiki.uino.com/book/thingjs-editor/6710940608b62209a0b23d12.html)

### 9\. 内置事件

覆盖编辑器状态、插件管理和面板操作等常见场景，开发者可通过监听这些事件，实现功能扩展。

详细文档：[内置事件](https://wiki.uino.com/book/thingjs-editor/66f11bdc3b407c4754cd543e.html)

### 10\. 插件调用

编辑器支持插件之间相互调用，实现插件间的高效协作。

详细文档：[插件调用](https://wiki.uino.com/book/thingjs-editor/67120f6fef1bf7cc7623f1d4.html)

### 11\. 插件参数

插件支持配置和读取参数，分为编辑时参数和运行时参数。

详细文档：[插件参数](https://wiki.uino.com/book/thingjs-editor/66f11c793b407c4754cd5485.html)

### 12\. 子编辑器

支持在独立窗口中加载子编辑器，为特定任务提供专属界面。子编辑器与主编辑器完全隔离，支持插件依赖和事件通信。

详细文档：[子编辑器](https://wiki.uino.com/book/thingjs-editor/66f11d213b407c4754cd54d2.html)

### 13\. 国际化

内置简洁高效的国际化方案，支持多语言切换。

详细文档：[国际化](https://wiki.uino.com/book/thingjs-editor/66f11c2a3b407c4754cd5459.html)

### 14\. 项目设置

支持存储和读取项目的自定义数据，这些数据可以在多个插件间共享。

详细文档：[项目设置](https://wiki.uino.com/book/thingjs-editor/6711d49e08b62209a0b24172.html)

### 15\. 动态生成

编辑器会实时监听插件文件内容的变化，动态生成需要的文件。

详细文档：[动态生成](https://wiki.uino.com/book/thingjs-editor/66f11bc73b407c4754cd542e.html)
