本文介绍了在编辑器中对象的自定义组件开发，包括创建、查看、编写和挂载等方法。您可以通过组件管理器或项目文件夹查看组件，创建组件时需指定名称、标题和位置。组件代码通过VSCode编辑，支持属性和方法导出。组件可在对象属性面板上挂载并修改属性，或在蓝图中调用。组件适用于主编辑器和预制件编辑器。

关于 ThingJS 中，对组件的概念以及功能介绍，请参考开发指南中的 [组件](https://wiki.uino.com/d/66f67ee83b407c4754cd64df.html) 部分。

# **查看组件**

两种查看方法：

1\. 菜单栏 → 开发 → 自定义组件 通过 UI 界面查看；  
2\. 打开项目目录，查看 `plugins/.common/components` 或`plugins/<插件名称>/components` 目录；

# **创建组件**

![](https://pan.uino.cn/files/image/2024/9/8a21cdd653831df0e6bbe34a46a2b733.png)

* 组件名：组件的名称，用来生成代码
* 标题 ：组件的标题，用来友好的提示当前组件的名称
* 位置 ：组件存放位置，也就是 `plugins/.common/components` 或 `plugins/<插件名称>/components` 目录

# **代码编写**

创建完毕组件后，会自动通过 vscode 打开组件路径，可以看到默认生成的模板

```javascript
/**
 * @title MyComponent
 * @bpNode BPMyComponent
 */
export class MyComponent extends THING.Component {
  static className = 'MyComponent'

  // 特别注意：要导出的属性或方法如果和对象的属性或方法存在重名问题，就会覆盖原来的，可能间接导致程序运行出错
  // 需要导出的属性名称
  static exportProperties = ['speed', 'axis']
  // 需要导出的方法名称
  static exportFunctions = []
  // 构造函数
  constructor(param = {}) {
    super(param)
    this._speed = param['speed'] || 30
    this._axis = param['axis'] || 'X'
  }
  /**
   * @bpNode 旋转
   * @title 速度
   * @type {number}
   * @default 30
   * @range [0, 1000]
   * @step 1
   */
  get speed() {
    return this._speed
  }
  set speed(value) {
    this._speed = value
  }
  /**
   * @bpNode 旋转
   * @title 轴
   * @type {string}
   * @data ["X", "Y", "Z"]
   */
  get axis() {
    return this._axis
  }
  set axis(value) {
    if (['X', 'Y', 'Z'].includes(value)) {
      this._axis = value
    }
  }
  // 创建时回调
  onAwake(param) {
    this._speed = param['speed'] || 30
    this.axis = param['axis']
    // 常用成员：
    // this.app、this.camera、this.object;
  }
  // 启动时回调
  onStart() {
    this.object.style.color = '0xFF0000'
  }
  // 更新回调，如果不需要更新，为了提升性能应该把本方法删掉
  onUpdate(deltaTime) {
    this.object['rotate' + this.axis](this.speed * deltaTime)
  }
  // 组件销毁时调用一次 onDestroy 方法
  onDestroy() {
    console.log('destroy...')
  }
  // 加载场景时需要反序列化的数据
  onImport(data, options) {
    this.speed = data.speed || 30
    this.axis = data.axis
  }
  // 保存场景时需要序列化的数据
  onExport(options) {
    return { speed: this.speed, axis: this.axis }
  }
  /**
   * @param {string} [日志级别="log"]
   * @return {string} result
   * @bpNode 打印当前状态
   */
  logCurrentStatus(level = 'log') {
    var s = `Current Status: angles: ${this.object.angles}`
    console[level] && console[level](s)
    return s
  }
}
```

组件相关的知识可以从[这里获得](https://www.thingjs.org.cn/documentation/20_component)  
在编辑器中，对注释部分做了扩展，以便自动生成蓝图：

* @bpNode nodeName：将生成蓝图节点
* @title title: 标题，更加友好的显示名称
* @type {type} : 属性的类型，暂时只支持 number、string、boolean、object、vector3
* @default defaultValue：默认值
* @data enumValue：可选值，枚举类型，比如\["X", "Y", "Z"\]
* @param {type} name description: 函数的参数名称及类型
* @return {type} name： 函数的返回类型及名称

如果组件中的类含有 get、set 属性，比如示例模板中的 speed、axis，则可以在属性面板中设置组件属性的初始值

# **挂载组件**

选中对象后，刻在对象的属性面板中选择已编写好的组件，则组件会自动挂载到对象上，随后可以在界面中调整组件的参数。

# **预制件使用组件**

在预制件编辑器中使用组件，与主编辑器中的使用方法是一致的。但是需要注意的是，预制件子编辑器和主编辑器的插件设置是不同的。如果组件对某些插件有依赖，需要确保该插件在预制件编辑器中已启用。
