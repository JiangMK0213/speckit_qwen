插件参数分为**编辑时参数**和**运行时参数**，它们的定义和使用因场景而异。本指南详细介绍两种参数的配置、使用方式及相关操作。

## 编辑时参数

编辑时参数用于插件在编辑器内运行时的配置。这些参数在 `config.json` 中定义，格式与编辑器的 **属性面板** 配置类型一致。参数配置后，可在菜单【开发 -> 插件管理】中查看和修改。

### 示例：编辑时参数配置

```bash
{
  "name": "my-plugin",
  "params": {
    "param01": {
      "name": "参数01", 
      "type": "text", 
      "value": "test"
    },
    "param02": {
      "name": "参数02", 
      "type": "switch", 
      "value": true
    }
  }
}
```

当参数值发生修改时，编辑器会通过事件通知插件。

### 监听参数变化事件

```javascript
editor.on('plugin:change-params', (name, data) => {
  console.log(name)  // 插件名称
  console.log(data)  // { "param01": "test", "param02": true }
})
```

### 获取编辑时参数

编辑时参数可以在插件入口文件的 `onBeforeInstall` 和 `onInstall` 函数中获取，或通过插件实例的 `getParams` 方法获取。

### 示例：在入口文件中获取参数

```javascript
const onBeforeInstall = (params) => {
  console.log(params)  // { "param01": "test", "param02": true }
}

const onInstall = (params) => {
  console.log(params)  // { "param01": "test", "param02": true }
}
```

### 示例：通过插件实例获取参数

```javascript
const plugin = editor.getPlugin('plugin-name')  // 获取插件实例
console.log(plugin.getParams())  // { "param01": "test", "param02": true }
```

## 运行时参数

运行时参数用于插件在**脱离编辑器运行**（如预览运行）时的配置。这些参数在项目根目录的 `index.json` 文件中定义，并通过程序动态写入。

### 示例：运行时参数配置

```bash
{
  "scripts": [
    // ...
  ],
  "plugins": [
    {
      "name": "plugin-name", 
      "url": "plugins/.runtime/xxx/main.xxx.js", 
      "params": {
        "test": "123"
      }
    }
  ]
}
```

### 设置和获取运行时参数

编辑器提供了 `setRuntimeParams` 和 `getRuntimeParams` 方法来设置和获取插件的运行时参数。

#### 示例：设置运行时参数

```javascript
editor.setRuntimeParams('plugin-name', {
  test: 123
})
```

#### 示例：获取运行时参数

```javascript
console.log(editor.getRuntimeParams('plugin-name'))  // { "test": 123 }
```
