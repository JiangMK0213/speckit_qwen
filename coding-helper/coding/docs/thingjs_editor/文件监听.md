ThingJS Editor 提供了文件监听接口，插件可以实时监听系统目录下文件的变化，确保对文件操作的及时响应。这一功能可以用于同步外部资源、监控文件更新等场景，提升插件的自动化处理能力。

## 使用示例

### 1\. 定义文件监听的处理函数

监听到文件变化时，触发处理逻辑：

```javascript
const watchHandler = (event, filePath) => {
  console.log(`文件事件：${event}`, `文件路径：${filePath}`)
}
```

### 2\. 开启文件监听

通过 `editor.watch` 方法启动监听。

```javascript
const listenerId = editor.watch('path/to/dir', watchHandler, options)
```

* `**path/to/dir**`：需要监听的目录路径。
* `**watchHandler**`：文件变化时的回调函数。
* `**options**`：可选参数，用于配置文件监听行为，支持 [**chokidar**](https://www.npmjs.com/package/chokidar) 库的所有选项，如忽略规则等。

示例配置 options：

```javascript
const options = {
  ignored: /(^|[\/\\])\../, // 忽略隐藏文件
  persistent: true          // 持续监听
}
```

### 3\. 取消文件监听

通过 `editor.unwatch` 方法停止监听，避免多余的资源占用。

```javascript
editor.unwatch(listenerId)

listenerId：监听器的 ID，由 editor.watch 返回，用于标识并移除对应的监听器。
```
