编辑器提供了事件系统接口，支持插件之间的事件通信。插件可以使用这些接口**发送、监听、取消事件**，并在需要时传递数据。还支持异步事件处理和请求响应模式，使插件开发更加灵活高效。

## 基础事件操作

### 1\. 发送事件

开发者可以通过 `emit` 方法发送事件，并传递任意数据给其他插件或模块。

```javascript
// 发送事件
editor.emit('some-event', data)
```

### 2\. 监听事件

使用 `on` 方法注册事件监听器，并定义回调函数处理接收到的数据。

```javascript
const handler = (data) => {
  console.log('接收到的数据:', data)
}
// 注册事件监听
editor.on('some-event', handler)
```

### 3\. 取消事件监听

使用 `off` 方法取消事件的监听。

```javascript
editor.off('some-event', handler)
```

## 异步事件处理与请求响应

除了基础事件操作，编辑器还支持**等待响应的事件模式**，确保事件处理完成后返回结果。这对复杂的插件交互非常有用。

### 1\. 注册请求处理

使用 `handle` 方法注册事件的处理逻辑，并在回调函数中返回数据。该回调支持异步操作。

```javascript
<code>editor.handle('some-event', async (data) => {
  console.log('正在处理请求:', data)
  // 模拟异步处理
  await new Promise((resolve) => setTimeout(resolve, 1000))
  return { success: true, processedData: data }
})
```

### 2\. 发送请求并等待响应

使用 `invoke` 方法发送请求，并等待注册的处理函数返回结果。

```javascript
const response = await editor.invoke('some-event', { key: 'value' })
console.log('响应结果:', response)
```

> 注意：在调用 `invoke` 之前，必须使用 `handle` 方法注册相应事件的处理函数，否则会导致请求失败。
