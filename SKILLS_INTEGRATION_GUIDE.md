# 自定义技能与 spec-kit 和千问 code 集成指南

## 1. 集成目的

本文档旨在指导如何将 `.trea/skills` 目录中的自定义技能与 spec-kit 和千问 code 结合起来，以提高开发效率和代码质量。

## 2. 集成步骤

### 2.1 为自定义技能创建千问 code 命令

1. **创建命令文件**：在 `.qwen/commands/` 目录下为每个自定义技能创建对应的 `.toml` 命令文件。

2. **配置命令文件**：每个命令文件应包含以下内容：
   - `description`：命令的简短描述
   - `prompt`：命令的详细提示，包括执行步骤和上下文

3. **加载技能内容**：在命令的 `prompt` 中，通过加载技能的 `SKILL.md` 文件内容来提供技能的详细信息。

### 2.2 修改 spec-kit 脚本

1. **修改 check-prerequisites.ps1**：在脚本中添加对技能目录的检查，确保 spec-kit 能够识别和使用自定义技能。

2. **集成技能到工作流程**：在 spec-kit 的工作流程中集成技能的使用，例如在规划和实现阶段使用相关技能。

## 3. 使用方法

### 3.1 通过千问 code 使用自定义技能

1. **调用技能命令**：在千问 code 中使用 `/skill-name` 命令来调用对应的自定义技能。

2. **提供参数**：根据需要为技能命令提供参数，以获取更具体的指导。

3. **获取技能输出**：千问 code 会根据技能的内容生成详细的指导和建议。

### 3.2 在 spec-kit 工作流程中使用技能

1. **检查技能可用性**：运行 `check-prerequisites.ps1` 脚本时，会自动检查技能目录的存在。

2. **在规划阶段使用技能**：在制定实现计划时，参考相关技能的指导。

3. **在实现阶段使用技能**：在编写代码时，参考技能提供的最佳实践和规范。

## 4. 示例

### 4.1 调用 API 集成规范技能

```
/api-integration-spec
```

此命令会加载 API 集成规范技能的内容，并提供关于 RESTful API 设计、接口对接流程、错误处理和安全规范的详细指导。

### 4.2 调用园区开发指南技能

```
/campus-dev-guide
```

此命令会加载园区开发指南技能的内容，并提供关于 ThingJS 园区开发的详细指导，包括园区场景构建、层级控制、对象获取等核心功能。

## 5. 最佳实践

1. **定期更新技能内容**：根据项目需求和最佳实践的变化，定期更新技能的内容。

2. **扩展技能库**：根据项目的具体需求，创建和添加新的自定义技能。

3. **结合使用多个技能**：在开发过程中，结合使用多个相关技能，以获取更全面的指导。

4. **反馈和改进**：根据实际使用情况，不断反馈和改进技能的内容和集成方式。

## 6. 故障排除

### 6.1 技能命令不生效

- 检查命令文件是否存在于 `.qwen/commands/` 目录中
- 检查命令文件的格式是否正确
- 检查技能的 `SKILL.md` 文件是否存在且内容完整

### 6.2 spec-kit 无法检测技能目录

- 检查 `.trae/skills/` 目录是否存在
- 检查 `check-prerequisites.ps1` 脚本是否正确修改
- 确保技能目录中有有效的技能文件夹和 `SKILL.md` 文件

## 7. 结论

通过将自定义技能与 spec-kit 和千问 code 结合起来，可以提高开发效率，确保代码质量，同时为开发团队提供一致的指导和最佳实践。这种集成方式不仅可以满足当前项目的需求，还可以为未来的项目提供可复用的知识和经验。
