编辑器提供了历史记录功能。通过 `command` 接口执行的命令会被自动保存到历史记录中，从而支持撤销和重做操作。开发者可以将复杂的操作封装为命令，实现灵活的回退与恢复。

## 执行命令

使用 `command.execute()` 方法执行命令，并传入包含 `execute` 和 `undo` 两个函数的对象。这些函数分别定义了命令的具体执行逻辑和撤销逻辑。

### 示例：移动对象命令

```javascript
// 根据 id 移动对象到新位置
function moveById(id, newPosition) {
  let object // 存储目标对象
  let originPosition // 记录原始位置

  // 执行命令
  editor.command.execute({
    execute: () => {
      // 获取对象并记录其原始位置
      object = THING.App.current.query(`#${id}`)[0]
      originPosition = [...object.position]
      // 设置对象的新位置
      object.position = newPosition
    },
    undo: () => {
      // 恢复对象的原始位置
      object.position = originPosition
    }
  })
}

// 移动 id 为 'box' 的对象到 [0, 1, 0]
moveById('box', [0, 1, 0])
```

## 撤销与重做

通过 `undo` 和 `redo` 方法，开发者可以实现命令的撤销和重做操作。

```javascript
// 撤销操作
editor.command.undo()

// 重做操作
editor.command.redo()
```

## 复合命令

传入命令数组到 `execute` 方法时，可以执行多个操作的复合命令，从而一次撤销或重做多个操作。

### 示例：执行复合命令

```javascript
editor.command.execute([
  {
    execute: () => {
      console.log('执行操作 1')
    },
    undo: () => {
      console.log('撤销操作 1')
    }
  },
  {
    execute: () => {
      console.log('执行操作 2')
    },
    undo: () => {
      console.log('撤销操作 2')
    }
  }
])
```

## 查看历史记录

编辑器维护两个栈：`undoStack` 和 `redoStack`，分别保存撤销和重做的命令记录。

```javascript
// 查看撤销栈
console.log(editor.command.undoStack)

// 查看重做栈
console.log(editor.command.redoStack)
```

## 控制编辑器的已修改状态

默认情况下，执行命令会影响编辑器的已修改状态。如果不希望影响状态，可以在执行命令时传入第二个参数 `false`。

### 示例：执行命令且不影响编辑器已修改状态

```javascript
function moveById(id, newPosition) {
  let object
  let originPosition

  editor.command.execute(
    {
      execute: () => {
        object = THING.App.current.query(`#${id}`)[0]
        originPosition = [...object.position]
        object.position = newPosition
      },
      undo: () => {
        object.position = originPosition
      }
    },
    false // 不影响编辑器已修改状态
  )
}
```
